事件流时序
1
system/init — 会话初始化，返回 session_id、tools 列表、model、permissionMode
↓
2
assistant — AI 决定调用 Read 工具，输出 tool_use content block
↓
3
user — CLI 自动执行工具，返回 tool_result（文件内容）
↓
4
assistant — AI 根据工具结果生成最终文本回复
↓
5
result — 会话结束，返回 duration、cost、usage 统计

关键发现

Stream-JSON 协议特性总结

1. 5 种事件类型: system(init/hook), assistant, user(tool_result), result
2. Tool call 完整生命周期: tool_use → 自动执行 → tool_result → AI 继续回复
3. 会话恢复: session_id 可用于 --resume 恢复上下文（解决历史问题！）
4. 成本/用量: total_cost_usd + modelUsage 精细到每个模型
5. 权限模式: plan 模式下工具自动执行，无需审批交互
6. child_process 即可: 不需要 PTY，stdin/stdout JSON 流足够
注意: 本次测试用 plan 模式（工具自动执行）。default 模式下会输出 permission_request 事件，需要通过 stdin 回应。
